# VN 的結局設計：終結的藝術

視覺小說作為分岐敘事媒體，其結局設計承載了獨特的敘事功能與哲學意涵。不同於線性媒體只有「一個結局」，VN 必須處理「多重終結」的存在論問題：當故事可以有多種收束方式時，「結局」的意義是什麼？

---

## 一、VN 結局的類型學

### 按敘事功能分類

**1. Good End（好結局）**

定義：主角與攻略角色達成圓滿狀態的結局。

**特徵：**
- 羈絆確立或深化
- 衝突得到解決
- 情感上的滿足感
- 通常是該路線的「目標狀態」

**設計問題：**
- 過於完美可能顯得虛假
- 「完美」的定義因角色而異
- 某些角色的 Good End 可能傷害其他角色

**2. Bad End（壞結局）**

定義：故事走向負面收束——死亡、分離、失敗、悲劇。

**功能：**
- 警示錯誤選擇的後果
- 增加「好結局」的價值感
- 探索故事的黑暗面
- 提供完整體驗的一部分

**設計頻譜：**
```
輕微挫折 ──── 痛苦分離 ──── 角色死亡 ──── 世界毀滅
   │              │             │              │
 可挽回         永久           不可逆          終末
```

**Bad End 的敘事價值：**
- 《Fate/stay night》的 Dead End 提供大量世界觀補充
- 《School Days》的 Bad End 成為作品標誌性元素
- 部分 Bad End 比 Good End 更具藝術衝擊力

**3. Normal End（普通結局）**

定義：介於 Good 與 Bad 之間的中性收束。

**特徵：**
- 沒有完全的幸福，也沒有徹底的悲劇
- 故事告一段落但有未解之處
- 常被視為「真實」或「現實主義」結局

**設計意圖：**
- 避免二元對立的簡化
- 留白與餘韻
- 某些故事的 Normal End 是最恰當的收束

**4. True End（真結局）**

定義：被設計者定義為「正史」或「完整」的結局。

**特徵：**
- 通常需要特定條件解鎖
- 揭示完整的世界觀或真相
- 整合多條路線的伏筆
- 敘事地位高於其他結局

**True End 的問題域（詳見後續章節）**

**5. Secret End / Hidden End（隱藏結局）**

定義：需要特殊條件觸發的非常規結局。

**類型：**
- 彩蛋性質（輕鬆、搞笑）
- 完全隱藏的真相
- 需要破關多次才解鎖
- Meta 層次的結局

**6. Open End（開放結局）**

定義：故事刻意不給出明確收束，留待玩家詮釋。

**特徵：**
- 拒絕提供答案
- 強調過程而非終點
- 可能引發玩家討論與二創
- 文學性較高但可能引起不滿

---

### 按解鎖機制分類

**自然分歧型**
- 結局由選項自然導向
- 玩家的選擇直接決定走向
- 最直覺的設計

**累積型**
- 好感度、數值累積決定結局
- 結局是過程的總結
- 選項與結局的關係不直接

**路線鎖定型**
- 需按特定順序攻略才能解鎖
- 某些路線被 lock 直到完成前置
- 引導玩家體驗「正確」順序

**Meta 條件型**
- 需要破關多次
- 需要觸發特定 flag 組合
- 有時需要查攻略才可能達成

**隱藏條件型**
- 條件完全不提示
- 依賴玩家實驗或社群分享
- 可能數年後才被發現

## 二、True End 的存在論問題

### 什麼是「真」結局？

**正史定義：** True End 是被創作者認定為「實際發生」的結局，其他結局是「可能性」或「if」。

**問題：** 這個定義在 VN 的脈絡下是否成立？

**分歧敘事的本體論：**
- 玩家體驗了多個結局
- 每個結局在體驗當下都是「真實」的
- 事後宣稱「只有這個是真的」是否否定了玩家的其他體驗？

### 「真」的幾種意涵

**1. 時間線正史**
- 如果故事有續作或跨媒體延伸
- 需要選定一個時間線作為「發生過的」
- Fate 系列的複雜正史問題

**2. 敘事完整性**
- True End 是唯一解答所有謎團的結局
- 其他結局「缺少資訊」
- 完整性而非正確性

**3. 作者意圖**
- 「這是我想說的故事」
- 其他結局是商業需求或額外服務
- 作者主義的立場

**4. 玩家共識**
- 社群認定的「應該」結局
- 可能與官方定義不同
- 集體詮釋的力量

### True End 設計的倫理問題

**對玩家時間的尊重：**
- 若 True End 否定其他路線，玩家在其他路線花的 20-50 小時是什麼？
- 「你喜歡的角色不是正史」的傷害

**解鎖條件的公平性：**
- 過於隱蔽的條件強迫查攻略
- 打破沉浸感
- 但過於簡單則失去「特殊」感

**內容量的分配：**
- True End 的獨佔內容太多 → 其他路線感覺「被騙」
- True End 的獨佔內容太少 → 解鎖後失望

## 三、Bad End 的敘事功能深掘

### Bad End 不是「失敗」

**遊戲化思維的陷阱：**
- 將 Bad End 視為「輸了」
- 急於重來「打到好結局」
- 錯過 Bad End 本身的價值

**Bad End 作為內容：**
- 《Fate/stay night》的 40 個 Dead End 各有獨特場景
- 某些作品的 Bad End 比 Good End 更有文學價值
- Bad End 可能是作品最令人難忘的部分

### Bad End 的類型細分

**1. 懲罰型 Bad End**
- 明顯錯誤選擇的直接後果
- 「如果你選了這個，你死了」
- 功能：教育玩家遊戲邏輯

**2. 探索型 Bad End**
- 提供主線無法獲得的資訊
- 「只有死了才知道的真相」
- 功能：獎勵探索精神

**3. 藝術型 Bad End**
- 悲劇作為美學選擇
- 比 Good End 更有衝擊力
- 功能：情感的淨化（catharsis）

**4. 諷刺型 Bad End**
- 玩家以為是 Good End 的結局被標記為 Bad
- 質疑「好」的定義
- 功能：促進反思

**5. Meta 型 Bad End**
- 打破第四面牆
- 角色意識到這是遊戲
- 功能：媒介反思

### 「刀」文化：Bad End 的情感消費

**中文圈的「刀」概念：**
- 用刀刺入心臟的比喻
- 主動尋求情感傷害
- 「這把刀我吃了」

**刀的消費心理：**
- 安全的痛苦體驗
- 情感的強度證明
- 社群的共同經歷

**設計者的責任：**
- 「刀」需要被 earned（鋪陳足夠）
- 無意義的悲劇 ≠ 好的刀
- 給玩家選擇接受刀的時機

## 四、結局數量的設計考量

### 多結局的優點

**1. 重玩價值**
- 「還有其他結局」的動機
- 延長遊戲壽命
- 增加單位成本的內容量

**2. 選擇的意義感**
- 「我的選擇真的有影響」
- 玩家 agency 的確認

**3. 角色公平性**
- 每個可攻略角色都有「贏」的可能
- 避免「官配」的排他性

**4. 市場期待**
- 「只有一個結局」在商業 VN 中罕見
- 玩家預期多結局

### 多結局的問題

**1. 資源分散**
- 15 個結局各分配 1/15 的資源
- vs 3 個結局各分配 1/3
- 深度 vs 廣度的取捨

**2. 正史問題**
- 前述 True End 的困擾
- 跨媒體延伸時的選擇壓力

**3. 品質不均**
- 「這個路線明顯是湊數的」
- 玩家批評「只有 X 路線值得玩」

**4. 探索疲勞**
- 「還要打第 8 輪才有最後一個結局」
- 重複內容的問題

### 結局數量的光譜

| 類型 | 結局數 | 代表作品 | 特色 |
|------|--------|----------|------|
| Kinetic Novel | 1 | Planetarian | 無選擇，純閱讀 |
| 精簡型 | 2-5 | 部分短篇 | 清晰的對比 |
| 標準型 | 5-15 | 多數商業作品 | 平衡點 |
| 膨脹型 | 15-40 | Fate/stay night | 細分 Bad End |
| 極端型 | 100+ | 某些模擬養成類 | 數值組合排列 |

## 五、「攻略順序」的隱性敘事設計

### 路線鎖定（Route Lock）

**定義：** 某些路線需要先完成其他路線才能進入。

**設計意圖：**
- 控制資訊揭露順序
- 避免「先玩 True 後玩其他」的反高潮
- 確保情感累積

**實作方式：**
- 直接鎖定（選項不出現）
- 間接引導（前置路線的選項更明顯）
- 隱藏條件（需要觸發特定 flag）

### 「推薦順序」的社群文化

**現象：**
- 幾乎每部知名 VN 都有「推薦攻略順序」
- 攻略站、wiki、討論串的標準內容
- 「不按這個順序會毀掉體驗」的警告

**順序的依據：**
- 劇情連貫性
- 情感累積曲線
- 爆雷風險
- 角色深度漸進

**例：《CLANNAD》推薦順序的爭論**
- 「先打風子線」vs「先打杏線」
- 最終線的體驗取決於之前積累
- 社群有多種「正確」順序

### 「推薦順序」的問題

**剝奪玩家自主性：**
- 「照著攻略走」是否還是「我的」體驗？
- 選擇的意義被外部指導消解

**首輪體驗的特殊性：**
- 第一次玩的發現感無法複製
- 「按推薦順序」是否最大化首輪體驗？
- 還是「隨自己喜好」有其價值？

**設計者的責任：**
- 是否應該設計成「任何順序都有好體驗」？
- 還是強制的順序是敘事設計的一部分？

## 六、結局後的延續：After Story 與 Fandisc

### After Story 的概念

**定義：** 主線結局後的額外敘事。

**代表作：《CLANNAD》的 AFTER STORY**
- 高中篇結束後的成人生活
- 結婚、工作、育兒
- 某些人認為是作品真正的核心

**After Story 的功能：**
- 滿足「後來怎麼了」的好奇
- 探索結局後的現實挑戰
- 角色更深層的發展

### Fandisc 的結局觀

**定義：** 本篇之外的補充內容（通常另外發售）。

**內容類型：**
- if 路線（本篇沒有的角色路線）
- 後日談（Good End 之後）
- 平行故事（其他角色視角）

**結局的無限延伸：**
- Fandisc 表明「結局不是終結」
- 商業動機 vs 故事完整性
- 「什麼時候才真正結束？」

### 結局的「消費」與「完結」

**消費角度：**
- 更多結局 = 更多內容 = 更多價值
- 玩家想要「更多」
- DLC、Fandisc 的商業邏輯

**藝術角度：**
- 故事需要終結才有意義
- 「大團圓」後的延續可能破壞餘韻
- 「知道何時停止」是創作的智慧

## 七、無結局的 VN：Kinetic Novel 與開放敘事

### Kinetic Novel 的單一路徑

**定義：** 沒有選項、沒有分岐的純閱讀 VN。

**代表作品：**
- 《Planetarian》
- 《Narcissu》
- 《Saya no Uta》（實際有選項但極少）

**單一結局的辯護：**
- 「這是作者想說的故事，就這一個」
- 選擇不一定增加價值
- 專注於一條路徑的深度

### 開放結局的設計

**特徵：**
- 故事在未解決狀態結束
- 刻意不提供答案
- 玩家自行詮釋

**功能：**
- 延續思考到遊戲之外
- 拒絕簡化的收束
- 文學性的選擇

**風險：**
- 「什麼啊，沒講完就結束了？」
- 被批評為偷懶或故弄玄虛
- 需要前期鋪陳支撐

### Loop 結構的「無終結」

**部分作品的結構：**
- 故事循環不斷
- 「結局」是下一輪的開始
- 真正的結局是打破循環

**例：《Steins;Gate》、《ひぐらしのなく頃に》**
- 時間循環作為敘事機制
- 「結局」的定義被重構
- 玩家需要理解「為什麼要循環」才能找到出口

## 八、結局作為商品：周邊與跨媒體的影響

### 「哪個結局是正史」的商業壓力

**動畫化時：**
- 必須選擇一條路線
- 或者嘗試整合多路線（通常失敗）
- 或者做多版本（罕見）

**遊戲續作時：**
- 需要選定一個世界線延續
- 其他路線的粉絲被排除
- 或者做平行續作

**周邊商品時：**
- 「官配」會影響周邊設計
- 某些角色的周邊銷量因此受限

### 結局的「所有權」

**誰決定哪個結局是「真的」？**

**創作者立場：**
- 「我是作者，我說了算」
- True End 就是正史

**玩家立場：**
- 「我的體驗是我自己的」
- 我選擇的結局對我而言就是真的
- 死亡作者（Death of the Author）的適用

**商業立場：**
- 「能延續 IP 的結局是正史」
- 實用主義的選擇

## 九、結局設計的心理學面向

### 結束的需求（Need for Closure）

**心理學概念：**
- 人類傾向尋求確定的答案
- 不確定性引發焦慮
- 「好結局」滿足 closure 需求

**VN 的挑戰：**
- 多結局本質上是「不確定」的
- 哪個才是「真的」？
- 玩家可能無法獲得完全的 closure

### 峰終定律（Peak-End Rule）

**心理學發現：**
- 人對體驗的評價主要取決於峰值（最強烈的時刻）和終點
- 結局的重要性被放大

**對 VN 設計的意涵：**
- 結局的品質對整體評價影響巨大
- 即使過程好，糟糕的結局可能毀掉一切
- 「爛尾」的心理機制

### 選擇的悖論（Paradox of Choice）

**現象：**
- 選擇太多反而降低滿足感
- 「我選的真的是最好的嗎？」
- 結局數量過多可能引發焦慮

**設計建議：**
- 結局數量有甜蜜點
- 或者設計成「所有結局都好」
- 減少 FOMO（Fear of Missing Out）
