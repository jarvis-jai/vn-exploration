# 養成遊戲（Raising Simulation）與視覺小說的交匯

養成遊戲（育成シミュレーション / Raising Simulation）是與視覺小說深度交織的鄰近類型。兩者在日本美少女遊戲市場有大量重疊，許多作品同時具備養成與 VN 元素。理解養成遊戲的設計語言，有助於把握 VN 類型邊界的模糊性。

---

## 什麼是養成遊戲

### 核心定義

**基本特徵：**
- 玩家扮演監護者/指導者/管理者角色
- 透過時間經過與選擇，養育/訓練/培養某個對象
- 對象可能是人物（女兒、學生、偶像）、怪物、寵物等
- 最終結局由養成過程中的數值與選擇決定

**與 VN 的根本差異：**
- VN 以「閱讀敘事」為核心
- 養成遊戲以「數值管理與成長」為核心
- 但兩者經常混合

### 時間結構

**週期性時間：**
- 通常以「週」「月」「年」為單位
- 每個時間單位可執行有限行動
- 行動影響數值成長

**有限遊戲長度：**
- 設定明確的終點（如「養育到 18 歲」）
- 結局根據最終狀態決定
- 可重複遊玩追求不同結局

## 類型起源：美少女夢工場

### 《美少女夢工場》（Princess Maker, 1991）

**基本資訊：**
- 開發：GAINAX
- 平台：PC-98，後移植多平台
- 發售：1991 年

**遊戲架構：**
- 玩家扮演退役勇者
- 從神處接受一名女嬰養育
- 每月安排女兒的活動（學習、打工、休息等）
- 10 年後（女兒 18 歲）根據數值決定結局

**創新之處：**
- 確立「養成對象為可愛角色」的範式
- 結合模擬經營與角色養成
- 多達 70+ 種結局（依版本）
- 開創了「美少女養成」這一遊戲類型

### 系列演進

**《美少女夢工場 2》（1993）：**
- 改進 UI 與平衡
- 加入更豐富的劇情事件
- 被認為是系列最佳作

**《美少女夢工場 3》（1997）：**
- 3D 模型嘗試
- 評價分歧

**《美少女夢工場 4》（2005）：**
- 回歸 2D
- 加入語音
- 多平台發行

**《美少女夢工場 5》（2007）：**
- 強化劇情元素
- VN 化傾向更明顯

### 設計遺產

**Princess Maker 確立的範式：**

1. **時間管理核心**
   - 有限的時間資源
   - 每個行動有機會成本
   - 玩家必須做出取捨

2. **數值驅動結局**
   - 多維度數值（體力、智力、魅力、道德等）
   - 數值組合決定結局
   - 鼓勵多周目嘗試

3. **隨機事件**
   - 非可控的遭遇
   - 增加變數與驚喜
   - 打破最佳化的可預測性

4. **角色成長視覺化**
   - 外觀隨數值變化
   - 成就感具象化

## 養成遊戲的主要支脈

### 女兒養成系（父女向）

**範式：**
- 玩家 = 父親/監護者
- 養成對象 = 女兒
- 情感基調 = 親情（但常有曖昧）

**代表作品：**
- 美少女夢工場系列
- 《True Love ～純愛手帖～》（1995）
- 部分《暮蟬悲鳴時》系作品

**設計特徵：**
- 長時間跨度（年為單位）
- 強調保護與培養
- 結局常涉及女兒的職業/婚姻/命運

### 偶像養成系

**範式：**
- 玩家 = 製作人/經紀人
- 養成對象 = 偶像/藝人
- 情感基調 = 職業關係（常發展為戀愛）

**代表作品：**
- 《偶像大師》（THE IDOLM@STER, 2005-）
- 《Love Live! 學園偶像祭》（2013-）
- 《Ensemble Stars!》（2015-）
- 《BanG Dream!》（2017-）

**設計特徵：**
- 數值與節奏遊戲結合
- 社群元素（與其他玩家競爭/合作）
- 大量可養成角色
- 課金深度高

### 戀愛養成系

**範式：**
- 玩家 = 主角本身
- 「養成」的是與攻略對象的關係
- 傳統意義上的「戀愛模擬」

**與純 VN 的界線模糊：**
- 有數值系統 = 養成要素
- 無數值、純選擇 = 接近純 VN
- 大量作品在兩者之間

**代表作品：**
- 《心跳回憶》（ときめきメモリアル, 1994）
- 《純愛手帖》系列
- 《秋之回憶》（Memories Off, 1999-）

### 戰鬥養成系

**範式：**
- 養成對象 = 戰鬥單位（人物或怪物）
- 核心玩法 = 養成 + 戰鬥
- 敘事往往是次要的

**代表作品：**
- 《怪物農場》（Monster Rancher, 1997-）
- 《數碼寶貝世界》系列
- 部分《寶可夢》衍生作

**與 VN 的距離：**
- 通常較遠
- 但部分作品有強 VN 元素（如某些寶可夢衍生）

## 《心跳回憶》：養成與 VN 的關鍵交會點

### 基本資訊

- 開發：KONAMI
- 首作：1994 年
- 平台：PC-Engine，後移植大量平台

### 為什麼重要

**確立「戀愛養成遊戲」範式：**
- 將 Princess Maker 的時間管理系統
- 與「攻略多名女主角」的結構結合
- 創造了影響深遠的模板

**設計架構：**

1. **三年高中生活**
   - 固定時間框架
   - 每週/每月分配行動
   - 畢業時決定結局

2. **多角色攻略**
   - 同時存在多名可攻略角色
   - 行動選擇影響與各角色的好感度
   - 必須平衡或專注

3. **好感度系統**
   - 數值化的關係指標
   - 超過閾值觸發劇情事件
   - 最終告白成功與否取決於好感度

4. **約會事件**
   - 特定條件下觸發
   - 這部分接近 VN 的呈現方式
   - 對話、選項、劇情展開

### 對後世的影響

**「Tokimeki」模式的擴散：**
- 大量戀愛遊戲採用類似架構
- 成為「galge」的基本範式之一
- 與「純 VN」形成光譜

**養成要素的程度差異：**

| 類型 | 數值管理 | 時間分配 | 劇情比重 |
|------|----------|----------|----------|
| 重養成 | 核心 | 核心 | 較低 |
| 中間 | 存在 | 部分 | 中等 |
| 輕養成 | 簡化/隱藏 | 弱化 | 較高 |
| 純 VN | 無 | 無 | 核心 |

## 養成×VN 混合設計的光譜

### 重養成端：養成為主，敘事為輔

**特徵：**
- 大部分遊戲時間在數值管理
- 劇情作為獎勵/變化
- 多周目是預期的

**設計挑戰：**
- 如何讓數值系統有趣而不枯燥
- 如何在重複玩法中保持新鮮感
- 敘事如何不被數值系統割裂

### 中間地帶：兩者均衡

**特徵：**
- 養成階段與敘事階段交替
- 養成決定敘事分歧
- 兩者相互強化

**設計挑戰：**
- 平衡兩種玩法的節奏
- 避免任一方感覺冗餘
- 讓兩者在設計上真正整合

### 輕養成端：敘事為主，養成為輔

**特徵：**
- 核心是閱讀故事
- 養成元素簡化或隱藏
- 好感度系統可能在後台運作

**代表設計：**
- 《秋之回憶》系列（養成要素極輕）
- 部分現代「約會模擬 VN」
- 行動選擇影響路線，但不明確顯示數值

### Kinetic Novel 端：完全無養成

**特徵：**
- 沒有數值系統
- 沒有或極少選擇
- 純粹的閱讀體驗

## 養成機制的設計模式

### 數值設計

**常見數值類型：**

```
身體系：體力、健康、外貌
智力系：學業、知識、技能
社交系：魅力、好感度、人脈
心理系：壓力、疲勞、心情
特殊系：道德、信仰、特定劇情旗標
```

**數值互動：**
- 數值之間的影響（高壓力降低效率）
- 數值閾值觸發事件
- 數值組合決定結局

### 時間分配系統

**行動菜單設計：**

```
本週行動（選擇 1~3 項）：
- 上課（學業 +2）
- 運動（體力 +2，魅力 +1）
- 打工（金錢 +1000，壓力 +1）
- 約會（好感度 +?）
- 休息（壓力 -2）
```

**設計考量：**
- 每個行動的收益是否明確
- 機會成本是否有意義
- 是否存在「最佳策略」破壞平衡

### 隨機事件系統

**事件類型：**
- 固定事件（特定日期必定發生）
- 條件事件（滿足條件時觸發）
- 隨機事件（一定機率發生）

**敘事功能：**
- 打破重複感
- 提供驚喜與變化
- 傳達世界觀與角色性格

### 結局分歧設計

**數值決定型：**
- 最終數值組合對應結局
- 可計算、可預測
- 適合「目標導向」玩家

**旗標決定型：**
- 特定事件是否發生決定結局
- 更接近 VN 的設計
- 敘事連貫性較強

**混合型：**
- 數值決定大方向
- 旗標決定細節變化
- 最常見的設計

## 養成遊戲在中文圈的影響

### 「養成」作為類型標籤

**中文圈的特殊現象：**
- 「養成」成為獨立類型名
- 不完全對應日文「育成シミュレーション」
- 有時泛指任何有「培養關係」元素的遊戲

**用法範例：**
- 「這是一款養成遊戲」
- 「有養成要素」
- 「戀愛養成」（幾乎等於 dating sim）

### 台灣市場的養成遊戲傳統

**早期 PC 遊戲：**
- 受美少女夢工場影響
- 大量本土養成遊戲嘗試
- 《仙劍奇俠傳》（1995）也有養成元素

**Galgame 脈絡：**
- 日本 galgame 漢化引入
- 養成型戀愛遊戲有固定受眾
- 與純 VN 受眾有重疊但不完全相同

### 中國市場的發展

**手機遊戲浪潮：**
- 《戀與製作人》（2017）—— 乙女養成的商業成功
- 《光與夜之戀》（2021）
- 養成機制與課金模式的結合

**特徵：**
- 免費遊玩 + 課金
- 養成進度與付費掛鉤
- 社群競爭元素

## 養成遊戲的設計挑戰

### 「最佳化問題」

**問題：**
- 玩家傾向找出最優策略
- 一旦找到，遊戲變得機械化
- 養成的「選擇」變得沒有意義

**對策：**
- 增加隨機性
- 設計多種「等價最優」路線
- 讓「過程」而非「結果」成為樂趣

### 「重複疲勞」

**問題：**
- 多周目設計本意是增加遊玩時間
- 但重複相同流程令人疲倦
- 尤其當養成階段佔比大時

**對策：**
- 跳過/快進已讀內容
- 周目間的差異要夠大
- 縮短單次流程

### 「敘事割裂」

**問題：**
- 養成階段與敘事階段節奏不同
- 突然的數值管理打斷故事沉浸
- 或故事打斷養成的策略思考

**對策：**
- 讓養成本身有敘事意義
- 或讓敘事本身有養成功能
- 避免兩者生硬切換

### 「數值不透明」

**問題：**
- 隱藏數值讓玩家困惑
- 不知道行動是否有效
- 但完全透明可能破壞驚喜

**對策：**
- 部分透明（顯示大方向，隱藏細節）
- 事後揭示（結局時顯示數值）
- 足夠的回饋（即時反應但不精確數字）

## 現代趨勢：養成元素的輕量化

### 從顯性到隱性

**顯性養成（傳統）：**
- 明確的數值面板
- 玩家主動管理
- 「遊戲」感強

**隱性養成（現代趨勢）：**
- 數值在後台運作
- 玩家透過選擇間接影響
- 更接近「故事」體驗

### 從複雜到簡化

**複雜系統的退潮：**
- 現代玩家耐心降低
- 手機平台限制深度操作
- 「輕度遊戲」市場成長

**簡化的代價：**
- 失去策略深度
- 變得與純 VN 無異
- 養成的「身份」模糊

### 課金模式的影響

**養成進度的商品化：**
- 付費加速成長
- 付費獲得稀有角色
- 養成變成「付費檢定」

**對設計的影響：**
- 免費玩家的養成體驗被刻意設計得緩慢
- 「養成」變成「等待或付費」
- 原本的設計樂趣被商業考量扭曲

## 與 VN 設計的相互借鑑

### VN 從養成借鑑

**好感度系統：**
- 許多 VN 採用隱性好感度
- 選項影響數值，數值影響路線
- 源自養成遊戲的設計

**時間結構：**
- 部分 VN 採用日曆推進
- 每日選擇活動/見面對象
- 借用養成遊戲的框架

**多周目期待：**
- VN 的「全結局收集」
- 源自養成遊戲的設計哲學
- 重複遊玩是預期的

### 養成從 VN 借鑑

**敘事品質的提升：**
- 現代養成遊戲更重視劇情
- 角色刻畫技巧來自 VN 傳統
- 情感深度不只靠數值

**演出手法：**
- 立繪、CG、語音的運用
- VN 式的對話場景
- 事件的視覺化呈現
