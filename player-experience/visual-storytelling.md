# 視覺敘事與美術設計

視覺小說之所以稱為「視覺」小說，正是因為圖像在敘事中扮演的核心角色。然而，VN 的視覺語言有其獨特的文法與限制——不同於動畫的連續運動，不同於漫畫的分格節奏，VN 發展出一套以靜態圖像、漸變、與文字互動為基礎的視覺敘事系統。

---

## VN 視覺系統的組成要素

### 背景（Background / BG）

**基本功能：**
- 建立場景的空間感
- 告訴玩家「在哪裡」
- 定調氛圍（時間、天氣、環境）
- 提供視覺的「基底層」

**常見類型：**

| 場景類型 | 典型用途 | 設計考量 |
|----------|----------|----------|
| 教室 | 學園日常 | 需多種時段變體 |
| 走廊 | 過渡場景 | 簡潔但不單調 |
| 屋頂 | 告白、獨處 | 天空比例重要 |
| 主角房間 | 私密對話 | 個性化細節 |
| 咖啡廳 | 約會、會面 | 氛圍營造 |
| 街道 | 移動、偶遇 | 城市感/季節感 |
| 夜空/夕陽 | 情感高潮 | 色彩表達情緒 |

**背景的「版本」需求：**
- 時段變化：早晨 / 白天 / 傍晚 / 夜晚
- 天氣變化：晴天 / 陰天 / 雨天 / 雪天
- 狀態變化：正常 / 破壞後 / 節日裝飾

**製作方式：**
- **全手繪**：成本高，風格統一，品質可控
- **照片加工**：成本低，但風格可能與立繪不搭
- **3D 渲染**：可批量生成變體，但需要建模技術
- **AI 生成**：新興選項，版權與風格一致性有疑慮
- **混合**：部分手繪加工、部分照片處理

**常見問題：**
- 背景與立繪風格不統一
- 透視比例與角色不協調
- 重複使用導致視覺疲勞
- 變體不足導致時間感混亂

---

### 立繪（Character Sprite / Tachie）

**基本功能：**
- 角色的視覺呈現
- 表情與情緒的傳達
- 建立角色的「存在感」
- 與玩家建立視覺連結

**立繪的構成：**

```
┌─────────────────────────────┐
│         基底立繪            │
│    （姿勢、服裝、身體）      │
├─────────────────────────────┤
│         表情差分            │
│    （眼睛、眉毛、嘴巴）      │
├─────────────────────────────┤
│         附加層（可選）       │
│    （眼淚、臉紅、陰影）      │
└─────────────────────────────┘
```

**差分（Difference / Variation）的重要性：**

表情差分是 VN 視覺敘事的核心。同一基底立繪搭配不同表情，可以創造出豐富的情感光譜：

- **眼睛狀態**：睜開 / 半閉 / 閉眼 / 驚訝睜大 / 瞇眼笑
- **眉毛位置**：放鬆 / 皺眉 / 挑眉 / 悲傷下垂
- **嘴巴形狀**：微笑 / 驚訝張嘴 / 不悅抿嘴 / 說話中
- **附加效果**：臉紅 / 流淚 / 冒汗 / 青筋

**差分數量的考量：**

| 角色重要性 | 建議差分數 | 說明 |
|------------|------------|------|
| 主要攻略對象 | 15-30+ | 豐富的情感表達空間 |
| 重要配角 | 8-15 | 基本情緒覆蓋 |
| 次要角色 | 3-8 | 最低限度的變化 |
| 路人 | 1-3 | 可能只有一張 |

**立繪的「位置」語言：**

螢幕上立繪的位置傳達敘事資訊：

```
左側        中央        右側
┌───┬───────────────┬───┐
│ A │               │ B │  對話場景
└───┴───────────────┴───┘

        ┌───────┐
        │   A   │           單人聚焦
        └───────┘
        （中央放大）

左側    中央    右側
┌───┬───────┬───┐
│ A │   B   │ C │           多人場景
└───┴───────┴───┘
```

- **中央**：當前說話者 / 聚焦角色
- **左右對稱**：對話、對峙
- **偏移**：暗示權力關係或情緒傾斜
- **重疊**：親密、壓迫
- **離開畫面**：角色退場的暗示

**立繪的「大小」變化：**
- **放大（Zoom in）**：強調情感、親密感
- **縮小（Zoom out）**：距離感、群體場景
- **半身 vs 全身**：根據場景選擇

**立繪的動態效果（引擎支援）：**
- 震動（驚嚇、憤怒）
- 上下彈跳（開心、驚訝）
- 淡入淡出（登場、退場）
- 左右搖晃（猶豫、思考）
- 傾斜（俏皮、疑惑）

### CG（Event CG / 一枚繪）

**基本功能：**
- 標記重要劇情節點
- 提供立繪無法表達的構圖
- 作為「獎勵」滿足玩家
- 建立作品的視覺記憶點

**CG 與立繪的差異：**

| 特性 | 立繪 | CG |
|------|------|-----|
| 數量 | 大量使用 | 稀少珍貴 |
| 製作成本 | 中等 | 高 |
| 構圖 | 固定（正面/側面） | 自由 |
| 背景 | 分離 | 整合 |
| 重複使用 | 頻繁 | 通常一次 |
| 玩家期待 | 習以為常 | 驚喜感 |

**CG 的使用時機：**

**劇情高潮：**
- 重要轉折點
- 情感爆發場景
- 結局場景

**浪漫場景：**
- 告白
- 親密互動
- 約會高點

**視覺必要性：**
- 立繪無法表達的動作（擁抱、打鬥）
- 特殊視角
- 環境互動

**CG 數量的參考：**

| 作品規模 | CG 數量 | 備註 |
|----------|---------|------|
| 短篇（1-3小時） | 3-10 | 關鍵場景 |
| 中篇（5-15小時） | 10-30 | 每路線若干 |
| 長篇（20小時以上） | 30-80+ | 豐富的視覺獎勵 |

**CG 的「差分」：**
- 同一 CG 的不同版本
- 表情變化
- 服裝變化（尤其成人內容）
- 時段/光線變化
- 小細節差異

**CG Gallery 的設計意涵：**
- 解鎖機制創造收集動力
- 作為「通關證明」
- 讓玩家重溫喜愛場景
- 常作為額外內容銷售（設定集）

---

### UI/UX 作為視覺元素

**對話框設計：**

對話框不只是「放文字的地方」，它本身就是視覺設計：

- **透明度**：完全透明 vs 半透明 vs 不透明
- **形狀**：矩形、圓角、不規則
- **位置**：底部、側邊、全螢幕
- **裝飾**：邊框、花紋、角色頭像

**對話框風格與作品調性：**
- **簡潔現代**：扁平、無裝飾
- **奇幻/古風**：羊皮紙質感、繁複邊框
- **科幻**：發光效果、HUD 風格
- **恐怖**：扭曲、不穩定、會「壞掉」

**字體選擇的影響：**
- 易讀性是基本要求
- 但字體本身傳達調性
- 角色專屬字體可強化個性
- CJK 字體的選擇較受限

**選單與介面的世界觀整合：**
- 存檔畫面的視覺設計
- 設定選單的風格
- 標題畫面的氛圍
- 這些都是「沉浸感」的一部分

## 視覺敘事的技法

### 色彩的敘事功能

**場景色調與情緒：**

| 色調 | 情緒聯想 | 使用場景 |
|------|----------|----------|
| 暖橙/黃 | 懷舊、溫暖、夕陽 | 感人場景、回憶 |
| 冷藍/紫 | 悲傷、疏離、夜晚 | 孤獨、憂鬱場景 |
| 高飽和 | 活力、青春 | 日常喜劇 |
| 低飽和 | 壓抑、現實 | 沉重劇情 |
| 紅色強調 | 危險、激情 | 緊張、浪漫 |
| 綠色基調 | 自然、平靜 | 鄉村、回歸 |

**色彩的「變化」作為敘事工具：**
- 同一場景，色調改變 = 時間或情緒改變
- 漸變效果暗示轉折
- 突然色彩變化 = 震驚效果

**角色配色的意涵：**
- 角色的代表色通常是設計好的
- 髮色、服裝、眼睛色的組合
- 與其他角色的視覺區分
- 配色可暗示角色性格或命運

---

### 構圖與視線引導

**VN 構圖的限制：**
- 螢幕比例固定
- 立繪通常是固定姿勢
- 背景是靜態的
- 需要在這些限制中創造變化

**CG 構圖的自由：**

CG 是 VN 中唯一可以使用「電影式構圖」的地方：

- **Rule of Thirds（三分法）**：將視覺重點放在三分線交點
- **Leading Lines（引導線）**：用線條引導視線到焦點
- **Depth（景深）**：前景/中景/背景的層次
- **Framing（框中框）**：用環境元素框住主體

**視角選擇的意涵：**
- **俯視**：角色處於弱勢、被觀看
- **仰視**：角色處於強勢、被仰望
- **平視**：平等、日常
- **特寫**：情感強調、親密
- **遠景**：孤獨、渺小、環境壓迫

**視線方向：**
- 角色看向螢幕 = 對玩家說話
- 角色看向畫外 = 有其他對象
- 角色視線交會 = 角色間的連結
- 角色避開視線 = 迴避、羞澀、隱瞞

---

### 時間的視覺表達

**場景內時間流逝：**

VN 作為靜態媒介，如何表達時間？

- **背景漸變**：白天 → 傍晚 → 夜晚
- **光線變化**：窗戶透進的光改變角度
- **天空顏色**：作為時間指標
- **文字描述輔助**：「不知不覺已經傍晚了」

**季節與長期時間：**
- 背景的季節版本
- 服裝的季節變化
- 環境細節（落葉、積雪、櫻花）
- 年曆、日期顯示

**倒敘與回憶的視覺標記：**
- 泛黃/褐色調 = 過去
- 邊緣模糊/光暈 = 回憶
- 不同的 UI 框架 = 不同時間層
- 角色外觀變化（年輕版本）

---

### 靜態中的「動態感」

**文字顯示的節奏：**
- 文字出現的速度影響「時間感」
- 逐字出現 vs 整句出現
- 停頓的運用
- 與語音同步時的考量

**畫面效果（Screen Effect）：**

引擎常提供的視覺效果，用於在靜態圖像上創造動態感：

- **震動（Shake）**：衝擊、地震、驚嚇
- **閃爍（Flash）**：打擊、閃電、記憶閃回
- **淡入淡出（Fade）**：場景轉換、時間流逝
- **模糊（Blur）**：失焦、意識模糊
- **色彩濾鏡**：夢境、回憶、情緒
- **噪點/膠片效果**：復古、老舊感

**立繪動態的有限運用：**

即使立繪是靜態的，引擎可以做到：
- 位移（左右滑動、上下彈跳）
- 縮放（靠近、遠離）
- 旋轉（輕微傾斜）
- 透明度變化（漸隱漸現）

這些效果的**克制使用**很重要——過度使用會顯得廉價或分散注意力。

## 視覺與文字的互動

### 圖文關係的類型

**1. 圖文一致（Redundant）：**
- 文字說「她笑了」，立繪顯示微笑
- 最基本的用法
- 安全但可能冗餘

**2. 圖文補充（Complementary）：**
- 文字描述內心，立繪顯示外在
- 「雖然嘴上這麼說，但她的表情看起來...」
- 增加層次

**3. 圖文對比（Contradictory）：**
- 文字說一套，圖像顯示另一套
- 暗示角色在說謊或壓抑
- 高級技法，需要刻意設計

**4. 圖像主導（Image-led）：**
- 不用文字，讓圖像說話
- 沉默的 CG 場景
- 需要構圖能力

**5. 文字主導（Text-led）：**
- 黑畫面 + 純文字
- 特定場景效果（夢境、思緒）
- VN 有時會刻意「拿掉圖像」

---

### 「看不見」的敘事設計

**刻意不顯示：**

有時候，「不讓玩家看到」是設計選擇：

- **主角無臉/無立繪**：促進代入感
- **關鍵場景不給 CG**：留給想像
- **黑畫面**：暴力、親密、恐怖場景的迴避
- **延遲揭露**：角色外貌的懸念

**視覺的「遮蔽」：**
- 逆光剪影：身份未明的角色
- 陰影籠罩：威脅、神秘
- 畫外聲音：有角色但不出現

**文字暗示 vs 視覺展示：**
- 暴力場景：文字描述 vs 血腥 CG
- 親密場景：暗示 vs 明示
- 各有其效果與受眾考量

## 美術風格的選擇

### 日式與非日式的光譜

VN 的視覺風格，最基本的分野是「日式」與「非日式」：

**日式風格特徵：**
- 大眼睛、小鼻子、尖下巴
- 誇張的髮色
- 特定的「萌系」比例
- 受動漫/漫畫影響深

**西方 VN / OELVN 的風格探索：**
- 更多樣的美術風格
- 可能接近插畫、漫畫、甚至寫實
- 不一定追求「可愛」
- 反映不同的美學傳統

**風格選擇的影響：**
- 目標受眾的期待
- 與敘事內容的搭配
- 市場定位
- 美術資源的可得性

---

### 風格一致性的挑戰

**多人協作的風險：**
- 不同繪師的風格差異
- 立繪與 CG 風格不一
- 背景與角色的融合度

**解決方案：**
- 詳細的美術規範文件
- 統一的上色師
- 主繪師負責最終調整
- 風格測試與迭代

**AI 輔助的風格問題：**
- AI 生成圖像風格不穩定
- 與手繪部分的整合困難
- 「不對勁」但難以言說的違和感

---

### 寫實 vs 風格化

**寫實取向：**
- 接近照片的背景
- 人物比例接近真實
- 光影處理講究
- 可能用於嚴肅題材

**風格化取向：**
- 誇張的比例與表情
- 色彩可以更自由
- 明確的「這是虛構」訊號
- 適合奇幻、喜劇、萌系

**介於之間：**
- 大多數商業 VN 其實是混合
- 風格化的角色 + 相對寫實的背景
- 找到平衡點

## 成人內容的視覺設計（簡述）

### 18+ VN 的視覺考量

成人向 VN 有其獨特的視覺設計需求：

**CG 的比例分配：**
- 成人 CG 可能佔總 CG 的顯著比例
- 需要平衡「服務」與「劇情」
- 全年齡版本移除後的視覺缺口

**差分的複雜性：**
- 成人場景的差分數量龐大
- 服裝狀態、表情組合
- 製作成本高

**Steam 版與 Patch 的實務：**
- 審查標準的模糊地帶
- 外部 patch 的整合設計
- 不同版本的資產管理

---

## 美術製作的實務考量

### 外包與自製

**自製的優點：**
- 風格完全可控
- 迭代與修改彈性
- 長期成本可能較低

**外包的優點：**
- 可取得高品質專業美術
- 不需自己具備美術技能
- 專注於其他開發面向

**混合策略：**
- 核心立繪外包給專業繪師
- 背景使用資產或照片加工
- UI 自製或套模板

### 美術資源的預算配置

**假設性的預算分配參考：**

| 項目 | 小型專案 | 中型專案 |
|------|----------|----------|
| 立繪 | 40-50% | 30-40% |
| 背景 | 20-30% | 20-25% |
| CG | 20-30% | 30-40% |
| UI | 5-10% | 5-10% |

這只是粗略參考，實際比例依專案需求大幅變動。

### 時程考量

**美術通常是瓶頸：**
- 劇本可以獨自完成
- 程式可以獨自完成
- 但美術需要大量人工時數
- 外包時程難以完全控制

**階段性規劃：**
1. 概念設計 / 角色設計
2. 立繪基底繪製
3. 表情差分製作
4. 背景繪製
5. CG 繪製
6. UI 設計
7. 最終整合與調整

## 視覺敘事的失敗模式

### 常見問題

**立繪表情與文字情緒不符：**
- 「她憤怒地說」但立繪在微笑
- 通常是差分不足或實作疏忽
- 嚴重破壞沉浸感

**背景與劇情矛盾：**
- 文字說「夜深了」但背景是白天
- 「教室裡只有我們兩人」但背景有其他人
- 需要嚴格的腳本-美術對照

**視覺疲勞：**
- 同一背景使用過長
- 立繪變化太少
- 長時間沒有 CG 獎勵

**風格突兀：**
- 某張 CG 風格明顯不同
- 背景與角色「貼上去」的感覺
- 品質落差太大

### 玩家的視覺期待

**看過高預算作品後：**
- 對獨立作品的視覺有不合理期待
- 「立繪太少」「CG 太少」的抱怨
- 需要設定合理期待或在風格上取勝

**「AI 生成」的敏感：**
- 部分玩家排斥 AI 美術
