# 翻譯與在地化：跨越語言障壁的 VN

視覺小說是一種高度依賴文本的媒介。與動作遊戲或音樂節奏遊戲不同，VN 的核心體驗無法脫離語言文字。這使得翻譯與在地化成為 VN 跨文化傳播的關鍵瓶頸——也是一個充滿張力的創意與商業場域。

---

## 粉絲翻譯的黃金時代

### 早期狀況：無人問津的孤島

在 2000 年代以前，日本 VN 基本上是語言封閉的市場。除了極少數例外，VN 開發商從未考慮過英語化或其他語言版本。理由很簡單：

- **市場規模認知**：認為海外市場不存在或太小
- **成人內容審查**：許多 VN 含有成人要素，在西方法律環境下複雜
- **媒介偏見**：「只是エロゲ」的印象阻礙正當化
- **技術障礙**：早期引擎對雙位元組字符以外的語言支援不佳

結果是：想要體驗日本 VN 的非日語使用者，只能學日語——或者等待粉絲翻譯。

### 粉絲翻譯組的興起

**時間線：**
- **1990 年代末 - 2000 年代初**：零星的個人翻譯嘗試
- **2004-2010**：粉絲翻譯的「爆發期」
- **2010-2015**：成熟期，大型計畫完成
- **2015 至今**：與官方翻譯並存，轉型期

**重要的翻譯組與社群：**

| 組織/個人 | 代表翻譯作品 | 特徵 |
|-----------|--------------|------|
| mirror moon | Fate/stay night, Tsukihime | TYPE-MOON 作品的核心翻譯者，定義了英語 VN 社群的基礎詞彙 |
| Amaterasu Translations | Cross†Channel, Steins;Gate（非官方） | 高品質翻譯著稱，對敘事技巧的尊重 |
| TLWiki | 多部作品的協作翻譯 | Wiki 式開放協作模式 |
| No Name Losers (NNL) | Muv-Luv 系列（初期） | 長期計畫的典型 |
| Doki Fansubs→Doki Translations | 多部作品 | 動畫字幕組轉型 VN 翻譯 |

### 粉絲翻譯的文化特徵

**動機：**
- 純粹的熱愛與分享欲
- 學習日語的實踐場域
- 社群認同與聲望
- 「讓更多人體驗好作品」的使命感

**品質光譜：**

粉絲翻譯的品質差異極大：

1. **頂級翻譯**
   - 譯者精通雙語，理解敘事技巧
   - 有編輯（Editor）、校對（QC）、測試（Tester）等分工
   - 翻譯時間可達數年
   - 部分品質不輸或超越官方翻譯

2. **中等翻譯**
   - 基本傳達劇情
   - 偶有翻譯錯誤或不自然表達
   - 缺乏編輯潤飾

3. **機翻或劣質翻譯**
   - 依賴機器翻譯後微調
   - 大量錯譯、誤譯
   - 閱讀體驗極差
   - 有時對作品造成傷害

**著名的品質案例：**

- **Fate/stay night (mirror moon)**：成為西方 Fate 粉絲群的基礎。翻譯選擇（如「Servant」「Noble Phantasm」等術語）被官方沿用
- **Muv-Luv Alternative 早期翻譯**：多個團隊接力，品質起伏
- **Ever17 (TLWiki)**：敘事詭計的翻譯處理困難

### 法律灰色地帶

粉絲翻譯在法律上是明確的著作權侵權——未經授權的衍生作品。但日本 VN 業界對此態度微妙：

**默許的原因：**
- 沒有官方翻譯計畫時，粉絲翻譯不構成市場競爭
- 粉絲翻譯實際上培養了海外市場
- 追訴成本高於收益
- 部分開發者個人同情

**緊張時刻：**
- 當官方翻譯啟動時，粉絲翻譯組通常自行下架
- 少數案例中開發商/發行商發出 C&D（停止通知）
- JAST、MangaGamer 等早期嘗試有時與粉絲翻譯發生衝突

**灰色地帶的潛規則：**
- 已有官方翻譯的作品，粉絲翻譯被視為不當
- 官方未計劃翻譯的作品，粉絲翻譯被默許
- 補丁形式（需要玩家擁有原版）比獨立發布更被接受

## 官方翻譯的崛起

### 早期先驅

**日本公司的嘗試：**
- **Himeya Soft / G-Collections（2000 年代初）**：早期的英語成人 VN 發行嘗試
- **Hirameki International**：DVD-PG 格式的嘗試，商業上失敗
- **Peach Princess**：JAST USA 的前身/關聯品牌

**困境：**
- 市場規模小
- 翻譯成本相對銷量過高
- 成人內容的零售通路受限
- 實體發行的物流挑戰

### JAST USA 與 MangaGamer

**JAST USA：**
- 美國最早的專業 VN 發行商之一
- 創辦人 Peter Payne 是日本 VN 文化的早期傳播者
- 發行模式從實體郵購演進到數位下載
- 代表作品：Steins;Gate 系列、Nitroplus 作品、Muv-Luv 系列

**MangaGamer：**
- 2008 年成立，純數位發行起家
- 與多家日本中小開發商建立關係
- 代表作品：Higurashi、Umineko、Fata Morgana
- 嘗試過翻譯外包給英語社群成員

**兩者的差異：**
- JAST 偏向大作與長期計畫
- MangaGamer 產量較高，涵蓋更多中小型作品
- 兩者都有翻譯品質波動的批評

### Steam 與市場轉型

**2014-2016 的轉捩點：**

Steam 開始接受 VN 上架（雖然成人內容仍受限），徹底改變了西方 VN 市場的格局：

1. **曝光度**：VN 能被主流玩家發現
2. **付款便利**：不再需要專門的成人內容商店
3. **願望清單與促銷**：整合進主流遊戲消費流程
4. **合法性**：在 Steam 購買 = 正當消費

**結果：**
- 日本開發商開始認真考慮英語化
- 更多第三方發行商進入市場
- Kickstarter 成為翻譯計畫的募資平台
- 「官方翻譯」不再是稀有事件

### 當代發行商生態

| 發行商 | 特徵 | 代表作品 |
|--------|------|----------|
| JAST USA | 老牌，品質波動 | Muv-Luv, Steins;Gate 系列 |
| MangaGamer | 產量高，範圍廣 | Higurashi, Umineko |
| Sekai Project | Kickstarter 募資模式先驅 | CLANNAD, Fault, Nekopara |
| Spike Chunsoft | 大廠直接進入西方市場 | Zero Escape, Danganronpa |
| NIS America | JRPG 發行商跨足 VN | 部分視覺小說相關作品 |
| NekoNyan | 較新進入，專注萌系 | Making*Lovers, Sabbat of the Witch |
| Aksys Games | 主要是乙女遊戲 | Collar×Malice, Code:Realize |
| IFI (Idea Factory International) | 乙女遊戲專門 | Otomate 作品 |

## 翻譯的挑戰：不只是語言

### 日語特有的敘事技巧

**人稱與語氣的問題：**

日語的人稱代詞和語氣微妙地傳達角色個性：

- 「俺」vs「僕」vs「私」vs「わし」——男性第一人稱的差異在英語中難以傳達
- 「～ですわ」「～なのよ」「～だぜ」等語尾在英語中沒有對應
- 敬語階層（タメ口、敬語、謙讓語）的豐富層次

**翻譯策略：**
- 語氣副詞補償（"I say," "ya know," "indeed"）
- 方言對應（關西腔 → 美國南方腔？有爭議）
- 有時直接放棄，接受損失

**雙關語與文字遊戲：**

VN 常有基於日語同音異義或漢字視覺的文字遊戲：

- 角色名字的雙關意義
- 劇情關鍵的文字謎題
- 搞笑場景的語言幽默

這些往往是「不可譯」的，翻譯者必須：
1. 嘗試創造對應的英語文字遊戲（可能偏離原意）
2. 直譯並加註解（打斷閱讀）
3. 完全改寫（可能失去原味）
4. 承認損失（誠實但遺憾）

**文化典故：**

日本 VN 充滿對日本流行文化、歷史、文學的引用：

- 動畫/漫畫梗
- 特攝/偶像文化
- 日本史與傳說
- 日本都市傳說
- 學校文化（體育祭、文化祭、部活）

### 成人內容的處理

**「全年齡版」的製作：**

許多日本 VN 原本含有成人場景。在西方（尤其是 Steam）發行時面臨選擇：

1. **製作「全年齡版」**：刪除或重寫成人場景
2. **分離發售**：Steam 版乾淨，官網賣「18+ patch」
3. **完全不發行成人內容**

**問題：**
- 有些成人場景對劇情有意義
- 「patch」的灰色地帶
- 社群對「審查」的反感
- 平台政策的不透明

### 翻譯團隊的組成

專業 VN 翻譯通常需要：

| 角色 | 功能 |
|------|------|
| 翻譯（Translator） | 日→英的核心轉換 |
| 編輯（Editor） | 確保英語自然流暢 |
| 校對（TLC, Translation Checker） | 確認翻譯準確度 |
| QA/測試 | 在遊戲中確認文字顯示、bug |
| 程式（Hacker/Engineer） | 提取/植入文字，處理引擎問題 |
| 項目經理 | 協調時程與溝通 |

**常見問題：**
- 翻譯者精通日語但英語寫作平庸（需要強力編輯）
- 編輯不懂日語，可能「改錯」
- 溝通不足導致不一致
- 時程壓力下品質犧牲

## 翻譯風格的爭議

### 「忠實」vs「自然」的光譜

VN 翻譯社群對「好翻譯」的標準有長期爭論：

**「忠實派」（Literal / Faithful）：**
- 盡可能保留原文結構
- 保留日本文化元素（敬稱如 -san, -kun, -chan）
- 不過度「西化」
- 寧可略顯生硬也不改動原意

**「自然派」（Liberal / Localization）：**
- 優先英語的流暢自然
- 文化元素適度轉化（-san → "Mr."？直接省略？）
- 為目標語言讀者優化體驗
- 接受創造性改寫

**實際案例爭議：**

- **敬稱保留問題**：很多讀者已習慣 -san, -senpai 等用法，但也有人認為這是「半吊子翻譯」
- **食物名稱**：「おにぎり」是 "onigiri" 還是 "rice ball"？（經典的 Pokémon "jelly donut" 爭議）
- **日本學校獨有概念**：「文化祭」「修學旅行」「部活」如何處理？

### 特定翻譯的著名爭議

**Fate/stay night "People die when they are killed"：**
- 字面上荒謬的台詞
- 但在脈絡中有其意義（主角之前有不死能力）
- 成為迷因，但也是討論翻譯脈絡重要性的案例

**Steins;Gate "El Psy Kongroo"：**
- 原文故意的偽拉丁文
- 翻譯需要傳達「故意錯誤」的感覺
- 音譯/意譯的選擇

**Umineko 的魔女台詞風格：**
- Beatrice 的傲慢語氣
- 日語的「高貴」語氣如何轉化
- 不同翻譯版本風格差異明顯

## 其他語言的翻譯生態

### 中文翻譯

**繁體中文市場：**
- 台灣有小規模官方代理市場
- 光譜為主要發行商
- Steam 繁中版需求上升
- 粉絲翻譯組存在（漢化組）

**簡體中文市場：**
- 中國大陸的「漢化組」傳統
- 早期以盜版 + 漢化為主
- 近年 Steam 中國區促進正版化
- 官方簡中版增加
- 審查問題影響發行

**中文翻譯的特殊挑戰：**
- 日語漢字與中文漢字的微妙差異
- 日本特有詞彙的處理
- 中日文化相近但仍有落差
- 繁簡差異不只是字形，也有詞彙習慣差異

### 韓文翻譯

- 韓國有獨立的 VN 翻譯市場
- 部分日本作品有韓語版
- 韓國國產 VN 也存在
- 文化與日本接近，某些元素較易傳達

### 其他語言

- **西班牙語**：拉美與西班牙市場，粉絲翻譯活躍
- **俄語**：東歐市場，粉絲翻譯傳統強
- **法語、德語**：歐洲語言，官方翻譯較少
- **葡萄牙語（巴西）**：新興市場

## 翻譯與原作「正典性」

### 翻譯版本是否「正統」？

部分玩家堅持「只有原版才是真正的體驗」，這涉及幾個層面：

1. **語言損失**：翻譯必然丟失某些元素
2. **聲優體驗**：日語語音 + 英文文字的割裂
3. **文化浸潤**：用母語閱讀缺少「異文化體驗」
4. **作者意圖**：原作者用日語思考和創作

**另一方的反駁：**
- 大多數人沒有條件學日語
- 好的翻譯能傳達核心體驗
- 「正統性」不應成為精英門檻

### 翻譯選擇影響後續詮釋

**術語的定型：**
- mirror moon 的 Fate 翻譯用語被官方採用
- 粉絲翻譯的術語選擇可能「鎖定」社群理解

**不同翻譯版本造成的社群分裂：**
- 讀不同翻譯版本的玩家，對同一作品可能有不同理解
- 翻譯更新時的「經典 vs 新版」爭議

## 技術層面的在地化挑戰

### 引擎與文字編碼

**早期問題：**
- 許多 VN 引擎假設日語環境
- 雙位元組字符（Shift-JIS）的限制
- 西方字母的字型缺失
- 文字換行邏輯差異（日語不需空格斷詞）

**現代改善：**
- Unicode 支援成為標準
- 主流引擎（Ren'Py, Unity）原生多語言
- 但舊作移植仍有挑戰

### 文字長度問題

日語通常比英語簡潔。翻譯後文字可能膨脹 1.5-2 倍，導致：

- 對話框溢出
- UI 需要調整
- 語音與文字長度不匹配

### 字型與排版

- 西方文字的 VN 字型選擇
- 可讀性 vs 風格
- 斜體、粗體等標記的使用

---

## 未解決的張力

**商業可行性 vs 品質：**
- VN 翻譯的利潤空間通常很小
- 壓低成本往往犧牲品質
- 「快速上市」vs「精雕細琢」

**粉絲翻譯的角色轉變：**
- 官方翻譯增加，粉絲翻譯的空間縮小
- 但官方品質不佳時，粉絲翻譯成為「替代」
- 舊作（無官方翻譯計畫）仍依賴粉絲

**機器翻譯的威脅與可能：**
- AI 翻譯品質快速提升
- 能否替代人工翻譯？
- VN 的文學性與機器翻譯的衝突
- 可能用於初稿但仍需人工潤飾
